Geoprocessing Tutorials

This page compiles practical workflows developed by Andressa using Google Earth Pro and QGIS. Each section documents a complete task—from data selection and layer creation to analysis and visualization— with a focus on environmental and land-use applications.

The goal is to serve as a visual and didactic reference for students, collaborators, and professionals interested in applying geospatial tools to real research and monitoring projects.

Mapping with Google Earth Pro

Objective: Compare land-use changes in a new allotment between 2008 and 2018 in São Miguel do Oeste–SC.

Step-by-step

  1. Select the target area (the allotment finalized in 2018).
  2. Draw a polygon to delimit the total area of the territory.
  3. Draw polygons for each block inside the allotment.
  4. Draw paths for the streets of the area.
  • Total area: 52,017 m².
  • Sum of all blocks: ≈31,941 m².
  • Street area (approx.): 52,017 − 31,941 ≈ 14,076 m².

Temporal analysis (2008 × 2018)

  1. Switch the imagery date to 2008 for the same territory.
  2. Using the 2018 total area polygon as reference (with lower opacity), delimit eucalyptus plantations.
  3. Delimit native forest area and adjust polygon overlap with eucalyptus and total terrain.
  4. Delimit the remaining area without vegetation (open field).
  • Eucalyptus: 40,353 m².
  • Native forest: 8,852 m².
  • Open field: 2,949 m².
  • Total territory (2008 composition): 52,154 m².

Mapping Areas with QGIS

Objective: Import the allotment mapped in Google Earth into QGIS and compute areas and lengths with proper symbology.

Workflow

  1. Import the KML/KMZ from Google Earth into QGIS.
  2. Install the QuickMapServices plugin and add an OSM Standard basemap.
  3. Calculate the area of each block polygon using the attribute table (field calculator).
  4. Open the statistics panel to obtain total area, mean, minimum and maximum values for blocks.
  5. Measure streets (line layer) and generate length statistics.

Symbology and data organization

  1. Adjust line and polygon styles to improve visualization (colors, thickness, transparency).
  2. Export non-quantitative features to new layers and rename attributes (e.g., QuadrasUR = residential blocks, QuadrasUM = mixed-use blocks).
  3. For quantitative data, apply graduated symbology (e.g., natural breaks / Jenks for block areas).
  4. For qualitative data, use categorized symbology to distinguish UR vs UM blocks.
  5. Add a Google Satellite layer and digitize additional polygons for surrounding trees and new features.
  6. Create a new line layer for the main road, digitize it and apply a distinct style.

The final map combines quantitative (area classes) and qualitative (land-use categories) information in a single, readable layout.

Land-Use Segmentation with QGIS (dzetsaka)

Objective: Create ALOS (land-use segments) from satellite imagery using semi-automatic classification in QGIS.

Preparation

  1. Download the required raster and vector files for the study area.
  2. Install the dzetsaka classification plugin (requires scikit-learn).
  3. Create a new polygon shapefile for training samples and add a text field named de>classe.

Training samples

  1. Digitize polygons over tree/forest areas and assign the class label (e.g., de>veg).
  2. Repeat for bare soil polygons (de>sex or similar class code).
  3. Create additional classes for agriculture types:
    • agr1 – greener agriculture (with active crops).
    • agr2 – lighter agriculture (without visible crops).
  4. Digitize polygons for pasture areas as another class.
  5. Update the attribute table so each training polygon has a consistent class label.

Classification and visualization

  1. Open dzetsaka, select the raster as input and the training shapefile as sample layer.
  2. Run the classification to generate a land-use raster.
  3. Customize colors for each class: trees (dark green), pasture (yellow), agriculture with crops (bright green), agriculture without crops (gray), bare soil (dark orange).

Comparing the classified raster with the original satellite image shows that splitting agriculture into “with crops” vs “without crops” significantly improves the interpretation of land use.

Intersection of Environmental Layers in QGIS

Guiding question: Which forest-cover polygons occur on soils of the Neosol type?

Layers used

  • Land use/cover layer (e.g., forest cover class).
  • Soil map layer with soil classes, including Neosols.

Steps

  1. Open both land-use and soil layers in QGIS.
  2. Filter the land-use attribute table to select only forest-cover polygons.
  3. Export the selected features to a new layer named, for example, de>solo_florestal.
  4. In the soil layer, filter the attribute de>PRIMNIVEL or equivalent to select Neosol units and export to a new layer de>neossolo.
  5. Open the Intersection tool (Vector > Geoprocessing Tools > Intersection).
  6. Set de>solo_florestal as the input layer and de>neossolo as the overlay layer, then run the algorithm.

The resulting intersection layer contains only polygons where forest cover and Neosol soils overlap, with attributes from both original datasets, allowing further area calculations and statistics.

Hypsometric, Slope and Shaded-Relief Maps

Objective: Generate terrain-analysis products (altitude, slope and hillshade) from a DEM using QGIS.

DEM preparation

  1. Download the elevation model (e.g., TOPODATA) for the study area.
  2. Load the raster DEM into QGIS and check its coordinate reference system.
  3. Adjust layer properties (color ramp, min/max values) to create a hypsometric map.

Profiles and hillshade

  1. Use the Profile Tool plugin (or Raster → Extraction → Profile) to generate topographic profiles along chosen transects.
  2. Open the Processing Toolbox and run a hillshade (shaded relief) algorithm from the “Raster terrain analysis” group.
  3. Zoom to areas of interest (e.g., river valleys) to inspect the relief representation.

Slope (clinographic map)

  1. Run the Slope tool (Raster terrain analysis) on the DEM to obtain slope values in degrees or percent.
  2. Apply a color ramp to highlight slope intervals (e.g., 0–19°, 19–38°, 38–57°, 57–76°).
  3. Combine hillshade and slope layers (using transparency) to produce a more intuitive terrain map.

Histograms with Custom Intervals

Objective: Build histograms with manually defined class intervals for geospatial or environmental variables.

General idea

  1. Export attribute data (e.g., areas, slope values, elevations) from QGIS to a table format (CSV or spreadsheet).
  2. Define custom class boundaries according to the analysis (for example, area size ranges or slope categories).
  3. Use a statistics tool (R, Python, Excel, etc.) to count the number of observations in each class.
  4. Plot the histogram using the predefined bins, ensuring that labels clearly reflect the custom intervals.

These histograms help interpret the distribution of quantitative variables extracted from geospatial layers, such as the dominance of certain block sizes or slope ranges in the study area.
